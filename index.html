AnonBlog is a simple web server for deploying anonymous static websites using Tor Rendezvous Points, otherwise known as "Hidden Services".

It is easy to deploy on most Linux distros.

Why

Standard web servers such as Apache, Nginx, and Lighttpd are old, large, and complex projects, that are not specifically made for Tor hidden services.

It is easy to mis-configure your server and end up with a compromised service. (This is a frequent problem with mod_status in Apache).

Once you have Kevin's public key in your key-ring, verify a release using this command: gpg --verify anonblog-beta-v1.1.3.tar.gz.sig

Replace the signature filename with the one appropriate to the release version.

Using It üíª

1. After downloading, make sure 'main.sh' is marked executable.

2. To start the server, simply do ./main.sh start

3. After starting for the first time, you can find your .onion address in anonblog/keys/

4. Put your website files in anonblog/site/

5. You're done! It's that simple!

You can use the included blog management script Bashblog to write and publish blogs, see its repo for more info.

To encrypt your onion's private key when not in use, set encryptPrivate to true in anonblog/config/abconfig

Tor provides built in (default off) authentication for hidden services, read more in the Tor manual and update your config/torrc accordingly.

Anonblog supports a limited amount of file extensions, see FILE-TYPES.txt

Building It Yourself üî®

It is highly recommended to compile the software yourself.

The source code is available on GitHub.

AnonBlog is made of 4 parts:

  • The main shell script - No need to compile
  • bbserver (the web server) - Compilable with GCC
  • BashBlog (the included static blog generator) - No need to compile
  • Tor - Includes a build script.

To build bbserver, simply do 'make bbserver'

To build Tor, download the source and follow the instructions on that page.

Warnings ‚ö†Ô∏è

AnonBlog is not magic, while we believe it is 'pretty good', and likely better than Apache/similar for static onion sites, you should never bet your life/liberty on software.

Your .onion's existence can be discovered by Tor nodes, keep this in mind. Update your AnonBlog's torrc with Tor's HiddenServiceAuthorizeClient feature if you do not want your onion to be publicly viewable.

Your ISP can see that you are connected to Tor, but not exactly what you do on it.

You should research correlation attacks used to uncover onion services, as these are not something AnonBlog can protect against.

You should scrub images/other files of metadata such as EXIF info before publishing.

To learn more about onion services, see here.
